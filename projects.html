NYC Yellow Taxi Trips: EDA + Statistical Testing

GitHub
  • Analyzed ~3.0M NYC Yellow Taxi trips (Jan 2024) to understand how trip volume, distance, and fares vary by time of day and day of week.
  • Performed data curation and preprocessing in pandas (type inspection, missing-value analysis, and row filtering) to create a clean, reproducible analysis dataset.
  • Built exploratory visualizations (daily and hourly trip trends; weekday/hour distributions) to surface usage patterns and guide follow-up analysis.
  • Tested whether trip distance differs by weekday using one-way ANOVA and followed up with Tukey HSD pairwise comparisons after filtering distance outliers.
  • Evaluated fare-related hypotheses including average fare by hour and fare differences by payment type (two-sample t-test), and summarized insights with clear plots and narrative explanations.

Iniquity+ Compiler Extensions: Arity, Rest Args, case-lambda, apply

GitHub
  • Extended a Racket-based compiler for a small functional language (Iniquity+) by implementing runtime arity checking for function calls, ensuring mismatched argument counts reliably signal errors.
  • Added support for rest-argument (variable arity) functions by generating code to pop extra stack arguments and construct a proper list bound to the rest parameter.
  • Implemented case-lambda-style multi-arity dispatch, selecting the correct clause at runtime based on the number of provided arguments (including mixed plain/rest clauses).
  • Built the apply form, compiling calls that take a list argument and push its runtime elements onto the stack before invoking the function, with error handling for non-list inputs.
  • Validated correctness using targeted interpreter-guided test programs covering arity mismatch, rest/list construction order, dispatch edge cases, and apply behavior.
